# Setting Up Real Wallets for Scout Analysis

## Current Issue

The wallets in your roster are **test/dummy wallets** generated by Scout's sample data loader. They don't exist on Solana mainnet because they're placeholder data for testing.

## Why Test Wallets?

The Scout service (`scout/core/analyzer.py`) currently uses `_load_sample_data()` which generates fake wallet addresses and metrics for testing purposes. This is intentional for development/testing, but not suitable for production.

## Finding Real Profitable Wallets

### Method 1: Use Solana Explorer

1. Go to https://explorer.solana.com/
2. Search for tokens you're interested in
3. Look at recent transactions
4. Identify wallets that made profitable trades
5. Add those addresses to your wallet list

### Method 2: Use Solscan

1. Go to https://solscan.io/
2. Browse token pages
3. Check "Top Traders" or "Recent Transactions"
4. Identify profitable wallets
5. Add addresses to Scout

### Method 3: Use Helius API Directly

Query Helius for recent profitable swaps:

```bash
curl "https://api-mainnet.helius-rpc.com/v0/transactions/?api-key=YOUR_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "type": "SWAP",
    "limit": 100
  }'
```

Then extract wallet addresses from the results.
